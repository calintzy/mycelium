# 하이브리드 검색 메모

검색 품질을 끌어올리는 가장 단순한 방법은 서로 다른 신호를 합치는 것이다.
의미 기반 검색(dense)과 키워드 기반 검색(sparse, BM25)은 서로의 약점을 보완한다.

## 왜 한 방식만으로는 부족한가
의미 검색은 단어가 달라도 뜻이 비슷하면 잘 찾지만, 고유명사나 정확한 코드 토큰,
제품명 같은 것을 놓치기 쉽다. 임베딩 공간에서 "BM25"와 "BM-twenty-five"가
가까울 보장이 없다. 반대로 키워드 검색은 정확히 같은 토큰을 잘 잡지만, 표현이
조금만 바뀌어도(동의어, 의역) 완전히 실패한다. 사용자가 "데이터가 사라지는 문제"라고
물었는데 문서에는 "데이터 유실"이라고 적혀 있으면 키워드 매칭은 0점을 준다.

## RRF로 합치기
Reciprocal Rank Fusion은 두 순위 리스트를 점수가 아니라 순위로 합친다.
각 문서의 점수는 1/(K + rank)의 가중합이며, K는 보통 60을 쓴다. 점수 스케일이
서로 다른 두 검색기를 정규화 없이 공정하게 합칠 수 있어 구현이 단순하다.
가중치에 둔감해 튜닝 부담도 적다. 이것이 dense 검색과 sparse 검색을 한 줄로
묶는 표준적인 방법이다.

## 한국어에서의 함정
한국어는 조사가 어절에 붙어 키워드 토크나이저가 "회복에"를 통째로 본다.
형태소 분석기를 쓰면 "회복" + "에"로 분리되어 질의 "회복"이 매칭된다.
공백 분리만 쓰면 한국어 키워드 검색은 영어보다 훨씬 약해진다.
