# .ai.md — mycelium 작업 가이드 (AI 에이전트용)

이 디렉토리를 수정하기 전에 읽는다. 기술 결정의 단일 진실은 `docs/DESIGN.md` (D-1~D-9).

## 이 프로젝트의 본질
마크다운 볼트를 코퍼스로 하는 하이브리드 검색 + RAG Q&A. 포트폴리오 + 실사용 도구.
공개 리포의 기본 코퍼스는 동봉된 합성 `sample_vault/`이며, 자기 노트는 `VAULT_PATH`로 지정한다.

## 절대 규칙 (위반 시 설계 위배)
- **검색은 직접 구현** (D-9): BM25(`rank_bm25`)+kiwi, RRF 융합을 직접 짠다. `langchain_community`(sunset)·`langchain_classic` import 금지.
- **한국어 BM25엔 kiwi 필수** (D-8): 공백 토크나이저 단독 금지. kiwi 실패 시 graceful fallback + 경고.
- **완전 로컬**: 임베딩·생성 모두 Ollama. 외부 네트워크 호출 금지(Anti-ISC). OpenAI/Claude API 호출 금지.
- **프라이버시 deny-by-default** (D-7): 공개 코퍼스는 `public: true` 노트만. 실볼트 데이터·인덱스는 커밋 금지(.gitignore 확인).

## 레이어 (의존 방향: interfaces → pipeline → adapters → core)
- `core/` — 도메인 모델(Document, Chunk, RetrievedNote, Answer), 설정. 외부 의존 최소.
- `adapters/` — embedding/llm/vectorstore 인터페이스. 로컬↔클라우드 교체는 여기서만.
- `pipeline/` — ingestion(로드·청킹·임베딩·적재), retrieval(하이브리드), generation(RAG).
- `interfaces/` — cli(typer), mcp_server(FastMCP). 입출력만.
- `eval/` — goldset + Hit@k/MRR.

## 모델
- 임베딩: Ollama `bge-m3` (1.2GB / 8K ctx) — 청크는 8K 토큰 미만 유지.
- 생성: Ollama `qwen2.5:14b` (9GB / 32K ctx).

## 흔한 실수
- `from langchain.retrievers import EnsembleRetriever` → 1.x에 없음. (애초에 D-9로 직접 구현)
- 청킹 시 헤더 메타 버리기 → 출처 표시·맥락에 필요하니 보존.
- Chroma 거리 메트릭 기본 L2 방치 → 정규화 임베딩엔 cosine 권장.
- 평가에서 grep에 MRR 적용 → grep은 무순위, Hit@k만.
