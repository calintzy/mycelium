## 실험 A: 소형 임베딩 모델 (plugin_default_bench.py)

### 실행 커맨드

```bash
KORQUAD_N_QUERIES=200 KORQUAD_N_CONTEXTS=2000 python benchmarks/plugin_default_bench.py
```

### 결과 요약

| 방식 | 모델 | 코퍼스 | Hit@5 | Hit@10 | MRR@10 | 표본 수 |
|------|------|--------|-------|--------|--------|--------|
| dense 단독 | all-minilm | KorQuAD 960단락 | 11.50% | 21.00% | 0.0761 | 200질의 |
| dense 단독 | all-minilm | 실볼트 426파일 | 54.5% | N/A | 0.2894 | 22질의 |

### 해석

- **KorQuAD (한국어 공개 벤치)**: all-minilm은 Hit@5 11.5% / MRR 0.076. bge-m3 dense 97.0% 대비 **−85.5%p** 차이. 영어 중심 소형 모델이 한국어 다중 단락 검색에서 극명하게 불리함을 확인.
- **실볼트 (개인 볼트)**: Hit@5 54.5% / MRR 0.289. bge-m3 100% 대비 **−45.5%p**. 볼트 특화 어휘(프로젝트명·고유명사)를 다국어 대형 모델 없이는 의미론적으로 잡지 못함.
- **KorQuAD 코퍼스 실제 규모**: datasets 라이브러리 미설치로 dev JSON 직접 다운로드 → validation 고유 단락 960개. 요청 2000단락 충족 불가(2000 요청 → 960 실제). korquad_bench.py와 동일한 동작.

### 프록시 가정 (한계)

- all-minilm은 Smart Connections 플러그인의 기본 모델과 동일하지만, 모든 플러그인이 이 모델을 쓰는 것은 아니다.
- Text Generator(OpenAI ada-002), Copilot(원격 API) 같은 플러그인은 훨씬 강한 모델을 쓴다.
- 따라서 이 결과는 "가장 가벼운 플러그인 기본값 상당"의 하한선이다.

### 해석

- **속도**: 질의당 평균 33초. Mycelium 로컬 dense 검색(수백ms)과 비교 불가 수준으로 느림.
- **비용**: 총 출력 토큰 48,219. claude CLI의 usage 입력 토큰이 `in=4~15`로 매우 낮게 잡히는 것은 CLI JSON 출력에서 실제 컨텍스트 토큰을 정확히 파싱하지 못했을 가능성이 있다. (실제 볼트 탐색 시 수천~수만 토큰 소비로 추정 — 비용 수치는 참고만 할 것)
- **카테고리별**: keyword(6) / paraphrase(6) / mixed(6) / graph(4) 모두 전승. 에이전틱 방식은 카테고리 편향 없음.

### 격리 확인

- 허용 도구: `Grep, Glob, Read` 전용 (볼트 수정 불가)
- 볼트 파일 수정: 0건 (읽기 전용 확인)
- 타임아웃: 질의당 최대 180초 (실제 최대 62초)

## 전체 해석 및 정직한 결론

### 에이전틱(B)이 이기는가?

- **정확도에서는 동등**: Hit@5 100%, MRR 0.924로 Mycelium bge-m3(볼트 100%, KorQuAD 97%)과 동등.
- **속도에서는 Mycelium 압승**: 33초/질의 vs 수백ms. 볼트 규모가 커지거나 질의가 많아질수록 에이전틱은 실용성이 없다.
- **비용**: 에이전틱은 API 토큰 비용 발생. Mycelium은 100% 로컬(무료).
- **즉각성**: Mycelium은 사전 인덱싱된 Chroma + BM25로 반응이 즉시. 에이전틱은 매 질의마다 볼트를 실시간 탐색.

### 소형 모델(A)이 비교 기준으로 타당한가?

- KorQuAD 11.5% vs bge-m3 97%의 격차는 "임베딩 모델 선택"이 검색 품질의 핵심 변수임을 확인.
- 실볼트 54.5% vs bge-m3 100%: 소형·영어 중심 모델로는 한국어 개인 볼트에서 절반만 찾는다.
- "옵시디언+소형 플러그인 조합"이 Mycelium보다 열세라는 가정은 이 측정으로 실증됨.

### 한계 및 표본 주의사항

- 볼트 goldset은 22문항으로 방향성 지표일 뿐 통계적 유의성이 없다.
- 에이전틱 in-token 수치(평균 8)는 CLI 파싱 한계로 실제 비용과 다를 수 있다.
- KorQuAD 실험은 datasets 미설치로 2000 → 960 단락 단축됨. bge-m3 기존 측정(2000단락)과 코퍼스 크기가 달라 직접 비교 시 주의 필요.
